Application Requirements - Group 10

The aim of the application is to allow the user to find parking lots in their surroundings as well as showing information about them, such as number slots, high occupation hours and geographical location.

Use Cases

ID - Use Case Name

US001 - View nearby parking lots

Description

As a user, I want to view nearby parking lots so I can drive to an area to park my car.

ID - Use Case Name

US002 - See information about a parking lot

Description

If I select a parking area the application will show me information about it, such as number of parking slots, type of parking slots or street.

ID - Use Case Name

US003 - Search average occupation by of the week and months of the year

Description

The application will allow you to select different days of the week and months of the year and it will calculate the average occupation of a parking area with previous data.

ID - Use Case Name

US004 - Filter by total number of slots

Description

As a user, I want to filter by the total number of spots so I can know the size of the parking lot.

ID - Use Case Name

US005 - Filter by average parking time

Description

As a user, I want to filter by average parking time so I can know how long cars stay in a certain parking lot.

ID - Use Case Name

US006 - Filter by area color

Description

As a user, I want to filter by area color so I can see where the area that suits me best is located.

ID - Use Case Name

US007 - Filter by time slot

Description

As a user, I want to filter by time slot so I can know the parking lot with the least number of cars in a specific hour.

Requirements

The application must follow these requirements:

  1. Show a map with the location of the different parking slots and their color.
  2. Show a dot that indicates the location of the closest parking meter to the parking lot.
  3. Show relevant information about the parking meters, for example its identification number.
  4. Show relevant information about the parking area, for example district, color and number of parking slots.
  5. Show relevant information for the user, for example average availability during the sun hours.
