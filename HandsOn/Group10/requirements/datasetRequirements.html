Dataset Requirements - Group 10

This file explains why the selected datasets satisfy the different requirements established for this assignment, according to the data license, format, belonging to the smart cities domain, documentation and the possibility to link the data to real-world entities.

Requirement 1: Data are in the smart cities domain

Our dataset gathers data from different parking areas of the city of Madrid displaying their location, average occupation and other characteristics. This dataset could help to solve parking problems in the city, so It will be included in the smart traffic domain.

Requirement 2: Data are available as a CSV file

Data is available to download as a csv file in the open data web of the city of Madrid.

Requirement 3: Data have an open license so it can be published

The data has an open license that allows us to use it for commercial and non commercial purposes. The specification of this license is in the next URL https://datos.madrid.es/portal/site/egob/menuitem.400a817358ce98c34e937436a8a409a0/?vgnextoid=b4c412b9ace9f310VgnVCM100000171f5a0aRCRD&vgnextchannel=b4c412b9ace9f310VgnVCM100000171f5a0aRCRD&vgnextfmt=default

Requirement 4: Data can be easily linked with real-world entities

The files contain the names of the different districts, neighborhoods and streets of Madrid which can be linked to the same locations in other datasets such as wikidata. Also the datasets contain the geographical coordinates of the parking meters, which can be used to link them with the same entities in other datasets.

Requirement 5: Documentation exists for the data

The data owner: Dirección General de Sostenibilidad y Control Ambiental, Subdirección General de Gestión de la movilidad from the Área de Gobierno de Medio Ambiente y Movilidad from the Madrid municipality provides a PDF file which explains the structure and content of the different datasets in the section named as "Documentación Asociada" in the next urls:
