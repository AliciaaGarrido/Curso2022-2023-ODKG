This file describe the dataSet Requirement

Matching Requirement

R1: Data in smart city domain

The data came from the city of Paris and represent public facilities, the position of available velib (city bikes) that is a part of smart transportation.
In a smart city one of the areas are "Sustainable Urban Mobility".
Transporting by bike is not only a good way to get around in a city, it is also a good way to reduce the use of cars and is therefore a sustainable option.
By approving the application for using this service, it can increase the use of the bikes.
Therefore we think that the degree of satisfaction for the first requirement given the chosen dataset is high.

R2: Data available as a CSV file

The dataset is available in an API, CSV file, online visualisation.
Therefore the second dataset is satisfied by the chosen dataset.

R3: Data have an open license so it can be published

The data are under Open Data Commons Open Database License. In short terms this means that we can freely use the the data,
as long as we attribute public use of the database, keeps the same license and keeps it open for others to use.
Therefore also the third requirement is satisfied by the chosen dataset, as we can publish it.

Data can be easily linked with generic real-world entities

R4: There are coordinate for all the datas, which makes it suitable to use this data in the application.
The fourth requirement is therefore satisfied, because it can easily be linked to a map.
